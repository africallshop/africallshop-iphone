Buddy configuration parameters like \doxyref{sip uri }{p.}{group__buddy__list_gaafd06626e9edf2c76ae85ec3351d38b2} or \doxyref{status publication }{p.}{group__buddy__list_ga9ab0c9222edcc0a8e25fb068aee88bc0} policy for this \doxyref{friend }{p.}{group__buddy__list_ga2f0508bab6d8ded431ce8b1a679a1522} are configurable for each buddy. \par
Here under a typical buddy creation\-: \par
 
\begin{DoxyCode}
LinphoneFriend* my\_friend=linphone_friend_new_with_addr(\textcolor{stringliteral}{"sip:joe@sip.linphone.org"}); \textcolor{comment}{/*creates friend
       object for buddy joe*/}
linphone_friend_enable_subscribes(my\_friend,TRUE); \textcolor{comment}{/*configure this friend to emit SUBSCRIBE message after
       being added to LinphoneCore*/}
linphone_friend_set_inc_subscribe_policy(my\_friend,LinphoneSPAccept); \textcolor{comment}{/* accept Incoming subscription
       request for this friend*/}
\end{DoxyCode}
 \doxyref{friends }{p.}{group__buddy__list_ga2f0508bab6d8ded431ce8b1a679a1522} status changes are reported by callback \doxyref{Linphone\-Core\-V\-Table.\-notify\-\_\-presence\-\_\-recv}{p.}{struct__LinphoneCoreVTable_afc93d0a51603a859fd30c0e38437d1c3} 
\begin{DoxyCode}
\textcolor{keyword}{static} \textcolor{keywordtype}{void} notify\_presence\_recv\_updated (\textcolor{keyword}{struct} \_LinphoneCore *lc,  
      LinphoneFriend *\textcolor{keyword}{friend}) \{
        \textcolor{keyword}{const} LinphoneAddress* friend\_address = linphone_friend_get_address(\textcolor{keyword}{friend});
        printf(\textcolor{stringliteral}{"New state state [%s] for user id [%s] \(\backslash\)n"}
                                ,linphone_online_status_to_string(
      linphone_friend_get_status(\textcolor{keyword}{friend}))
                                ,linphone_address_as_string (friend\_address));
\}
\end{DoxyCode}
 \par
Once created a buddy can be added to the buddy list using function \doxyref{linphone\-\_\-core\-\_\-add\-\_\-friend()}{p.}{group__buddy__list_ga3a8e9184320402082bf35226ae8d25b8} . Added friends will be notified about \doxyref{local status changes }{p.}{group__buddy__list_gadae69527e9b36e0f29bd80c30275874f} \par
 Any subsequente modifications to \doxyref{Linphone\-Friend}{p.}{group__buddy__list_ga2f0508bab6d8ded431ce8b1a679a1522} must be first started by a call to function \doxyref{linphone\-\_\-friend\-\_\-edit()}{p.}{group__buddy__list_ga254b01923c79947a5d23981d47639f40} and validated by function \doxyref{linphone\-\_\-friend\-\_\-done()}{p.}{group__buddy__list_ga6e28513bba28f948ee61341384a36958} 
\begin{DoxyCode}
linphone_friend_edit(my\_friend); \textcolor{comment}{/* start editing friend */}
linphone_friend_enable_subscribes(my\_friend,FALSE); \textcolor{comment}{/*disable subscription for this friend*/}
linphone_friend_done(my\_friend); \textcolor{comment}{/*commit changes triggering an UNSUBSCRIBE message*/}
\end{DoxyCode}


{\bfseries  Publishing presence status } \par
Local presence status can be changed using function \doxyref{linphone\-\_\-core\-\_\-set\-\_\-presence\-\_\-model()}{p.}{group__buddy__list_ga8b8f1c93a0c9a7d355b1899969058fb3} .New status is propagated to all friends \doxyref{previously added }{p.}{group__buddy__list_ga3a8e9184320402082bf35226ae8d25b8} to \doxyref{Linphone\-Core}{p.}{group__initializing_gaa5cf635b82dd338e7ee2dd3599d05f0a}.

{\bfseries Handling incoming subscription request} \par
 New incoming subscription requests are process according to \doxyref{the incoming subscription policy state }{p.}{group__buddy__list_ga9ab0c9222edcc0a8e25fb068aee88bc0} for subscription initiated by \doxyref{members of the buddy list. }{p.}{group__buddy__list_ga3a8e9184320402082bf35226ae8d25b8} \par
 For incoming request comming from an unknown buddy, the call back \doxyref{Linphone\-Core\-V\-Table.\-new\-\_\-subscription\-\_\-request}{p.}{struct__LinphoneCoreVTable_a728caa70b28680650669e7b341c11cb2} is invoked.
